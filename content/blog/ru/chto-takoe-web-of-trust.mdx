Доверие — это основа каждого взаимодействия в интернете. Когда вы читаете пост, подписываетесь на аккаунт или делитесь контентом, вы принимаете решение о том, кто заслуживает вашего внимания. Но как принимать такие решения в масштабе, без компании, которая решает за вас?

Именно на этот вопрос отвечает **Web of Trust**.

## Доверие без привратников

На большинстве платформ доверие централизовано. Компания верифицирует аккаунты, курирует ленты и решает, кого забанить. Это работает — пока не перестаёт. Цензура, предвзятость и непрозрачные алгоритмы разрушают то самое доверие, которое они призваны обеспечивать.

Web of Trust переворачивает эту модель. Вместо центрального авторитета **вы** являетесь корнем собственной сети доверия. Люди, на которых вы подписаны, — это ваш первый круг доверия. Люди, на которых подписаны они, формируют второй круг. И так далее.

<Callout type="tip">
Подумайте об этом как о реальной жизни. Вы доверяете друзьям. Друзьям друзей — в какой-то мере. Незнакомец в пяти рукопожатиях от вас? Вероятно, вы захотите каких-то доказательств, прежде чем ему доверять.
</Callout>

## Как это работает

Web of Trust отображает ваши социальные связи в направленный граф с измеримыми уровнями доверия:

- **1 шаг** — Люди, на которых вы подписаны напрямую. Максимальное доверие.
- **2 шага** — Друзья друзей. Умеренное доверие.
- **3 шага** — Два уровня удалённости. Пониженное доверие.
- **Более 3 шагов** — По сути, неизвестные для вашей сети.

Каждый шаг снижает оценку доверия. Но дело не только в расстоянии — количество **путей** тоже имеет значение. Если пять ваших доверенных контактов подписаны на одного и того же человека, он вызывает больше доверия, чем тот, к кому ведёт единственная цепочка.

## Истоки идеи

Web of Trust — не новая концепция. <WikiLink href="https://en.wikipedia.org/wiki/Phil_Zimmermann" extract="Creator of PGP, the most widely used email encryption software in the world, and a key figure in the public-key cryptography movement.">Фил Циммерман</WikiLink> представил эту концепцию в 1991 году вместе с <WikiLink href="https://en.wikipedia.org/wiki/Pretty_Good_Privacy" extract="An encryption program that provides cryptographic privacy and authentication for data communication, created by Phil Zimmermann in 1991.">PGP</WikiLink> (Pretty Good Privacy) — системой шифрования электронной почты. В <WikiLink href="https://en.wikipedia.org/wiki/Web_of_trust" extract="A decentralized trust model used in PGP and other systems, where trust is established through a network of mutually signed keys.">Web of Trust</WikiLink> PGP пользователи подписывали публичные ключи друг друга, подтверждая личность. Если вы доверяли Алисе, а Алиса подписала ключ Боба, вы могли обоснованно доверять и Бобу.

Идея была революционной: **доверие не требует центра сертификации. Оно может возникать из отношений.**

Nostr привносит эту концепцию в социальные сети. Вместо подписания криптографических ключей доверие распространяется через **подписки** — естественный и простой сигнал, который уже существует в протоколе.

## Почему Nostr нуждается в этом

Nostr — открытый протокол. Публиковать может каждый. Никто не может быть заблокирован на уровне протокола. Это его величайшая сила и самый большой вызов.

Без какой-либо фильтрации:

- Боты могут засыпать реле спамом
- Самозванцы могут копировать чью угодно личность
- Низкокачественный контент конкурирует на равных с ценными публикациями

Централизованные решения (списки блокировки, модерация администраторов) противоречат принципам Nostr. Web of Trust предлагает **децентрализованную альтернативу** — сеть доверия каждого пользователя действует как персонализированный, суверенный фильтр.

## Web of Trust на практике

С помощью [Nostr Web of Trust](/) ваше браузерное расширение строит локальный граф ваших социальных связей. Любое приложение Nostr может затем запрашивать этот граф для ответа на вопросы вроде:

- **Этот человек в моей сети?** Отфильтровать полных незнакомцев.
- **Как далеко он?** Приоритизировать более близкие связи.
- **Какова его оценка доверия?** Принимать взвешенные решения.

```javascript
// Проверить, находится ли публичный ключ в вашем Web of Trust
const trusted = await window.nostr.wot.isInMyWoT(pubkey);

// Получить расстояние в шагах
const distance = await window.nostr.wot.getDistance(pubkey);

// Получить нормализованную оценку доверия (от 0 до 1)
const score = await window.nostr.wot.getTrustScore(pubkey);
```

Результат: ваша лента отражает **ваши** суждения о доверии, а не алгоритма.

## Конфиденциальность по умолчанию

Система доверия, отслеживающая ваш социальный граф, могла бы стать кошмаром для конфиденциальности. Поэтому мы создали три режима:

- **Локальный режим** — Ваш граф полностью остаётся в вашем браузере. Ни один сервер его не видит.
- **Удалённый режим** — Запросы идут на наш oracle-сервер, но передаются только публичные ключи. Без контента, без метаданных.
- **Гибридный режим** — Локальное кэширование с удалённым запасным вариантом для неизвестных ключей.

Вы сами выбираете баланс между производительностью и конфиденциальностью.

## За пределами фильтрации

Web of Trust — это не только блокировка спама. Он открывает такие возможности, как:

- **Системы репутации** — Взвешивание контента по оценке доверия автора
- **Открытие нового** — Поиск ценных аккаунтов через доверенные пути
- **Коллективная модерация** — Сообщества могут определять общие границы доверия
- **Выбор реле** — Предпочтение реле, используемых доверенными контактами

Чем глубже ваш граф доверия, тем полезнее он становится.
