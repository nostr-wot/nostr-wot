Das Nostr-Ökosystem wächst rasant. Jeden Tag treten Tausende neuer Nutzer dem Netzwerk bei, erstellen Beiträge, teilen Inhalte und knüpfen Verbindungen. Doch mit diesem Wachstum kommt eine bekannte Herausforderung: **Wie trennt man Signal von Rauschen?**

## Das Problem offener Netzwerke

Offene, erlaubnisfreie Netzwerke sind gerade deshalb so mächtig, weil jeder teilnehmen kann. Aber diese Offenheit bedeutet auch den Umgang mit:

- **Spam und Bot-Konten**, die Ihren Feed überfluten
- **Identitätsdieben**, die sich als jemand anderes ausgeben
- **Minderwertigen Inhalten**, die wertvolle Diskussionen untergehen lassen

Traditionelle Lösungen setzen auf zentralisierte Moderation — ein Team entscheidet, was akzeptabel ist. Doch das widerspricht dem Zweck eines dezentralen Netzwerks.

## Web of Trust: Ein neuer Ansatz

Web of Trust verfolgt einen anderen Weg. Anstatt sich auf eine zentrale Instanz zu verlassen, nutzt es etwas, das Sie bereits besitzen: **Ihre sozialen Verbindungen**.

Das Konzept ist einfach:

1. **Sie vertrauen den Personen, denen Sie folgen**
2. **Diese vertrauen den Personen, denen sie folgen**
3. **Vertrauen fließt durch diese Verbindungen**

Wenn jemand 1 Hop entfernt ist (Sie folgen der Person direkt), genießt sie hohes Vertrauen. 2 Hops entfernt (ein Freund eines Freundes), mittleres Vertrauen. 3 oder mehr Hops? Wahrscheinlich Rauschen.

```javascript
// Prüfen, ob jemand in Ihrem Web of Trust ist
const inNetwork = await window.nostr.wot.isInMyWoT(pubkey);

// Die genaue Distanz ermitteln
const hops = await window.nostr.wot.getDistance(pubkey);

// Einen Trust-Score abrufen (0-1)
const score = await window.nostr.wot.getTrustScore(pubkey);
```

## So funktioniert es

Unsere Browser-Erweiterung erstellt einen lokalen Graphen Ihrer sozialen Verbindungen. Wenn Sie eine beliebige Nostr-App besuchen, kann diese den Graphen abfragen, um die Vertrauensstufe für jeden öffentlichen Schlüssel zu bestimmen.

<Callout type="tip">
Die Erweiterung funktioniert mit jeder Nostr-App, die die WoT-API implementiert. Keine Änderungen an Ihrem bevorzugten Client erforderlich!
</Callout>

### Datenschutz an erster Stelle

Wir haben das System mit Datenschutz als Kernprinzip entwickelt:

- **Lokaler Modus**: Alles bleibt in Ihrem Browser. Null externe Anfragen.
- **Remote-Modus**: Nur Abfragen öffentlicher Schlüssel — keine Inhalte, kein Tracking.
- **Hybrid-Modus**: Das Beste aus beiden Welten mit Caching.
